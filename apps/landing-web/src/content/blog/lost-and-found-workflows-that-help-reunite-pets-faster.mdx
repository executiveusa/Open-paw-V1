When a lost pet report comes in, time matters. The first 24–48 hours after a pet goes missing are the highest-probability window for reunification.

Most organizations handle lost/found reports through a combination of social media posts, paper intake forms, and individual email threads. This creates the conditions for things to fall through the cracks.

## The information problem

A lost pet coordinator is simultaneously managing:

- Incoming lost pet reports (from owners)
- Incoming found pet reports (from finders)
- Stray animals currently in the shelter or foster system
- Match reviews and follow-up calls
- Microchip database lookups
- Social media postings

None of these flows talk to each other. The result is that a match that should be obvious — the found dog in intake area B matches the lost dog reported last Tuesday — gets discovered by accident, if at all.

## What a clear workflow looks like

**Standardize intake.** Every lost pet report and found pet report goes into the same system with the same fields: species, breed, color, distinct markings, location, date, contact.

**Review for matches daily.** Rather than relying on memory or accident, assign someone to compare new reports against open cases each morning. This review is a scheduled task, not an ad hoc process.

**Connect to the animal records.** When a stray comes into the shelter, they should be checked against open found pet reports. The connection between the shelter's animal records and the lost/found database matters.

**Track communication.** Which reports have been followed up? Which potential matches were investigated and ruled out? Which cases are waiting for a callback?

**Close the loop.** When a pet is reunited, mark both the lost and found reports as resolved. This keeps the active queue clean.
